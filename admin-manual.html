CROSSPORT MSZ 施設予約システムの管理者向け運用マニュアルです。Google Calendar、Resend、Vercel を使った予約運用と障害対応の要点をまとめています。

1. 予約対象

Event Hall

営業時間 09:00 - 21:00

1時間単位。レイアウト変更オプションあり。

Meeting Room

営業時間 09:00 - 21:00

1時間単位。18:00以降は夜間料金。

Solo Booth

営業時間 09:00 - 18:00

5部屋を自動割当。画面上は残室数を表示。

2. 環境変数

GOOGLE_SERVICE_ACCOUNT_KEY
GCAL_ID_EVENT_HALL
GCAL_ID_MEETING_ROOM
GCAL_ID_SOLO_BOOTH_1
GCAL_ID_SOLO_BOOTH_2
GCAL_ID_SOLO_BOOTH_3
GCAL_ID_SOLO_BOOTH_4
GCAL_ID_SOLO_BOOTH_5
RESEND_API_KEY
RESEND_FROM_EMAIL
CANCEL_SECRET
SITE_URL

`GCAL_ID_EVENT_HALL` などの値には Google Calendar の「カレンダー ID」を入れます。カレンダー名ではありません。

`RESEND_FROM_EMAIL` は Resend で検証済みの送信元アドレスを入れます。

`SITE_URL` は本番では `https://crossport.cc` を設定します。

3. Google Calendar 設定

  1. Google Cloud のサービスアカウントを作成し、JSON キーを Vercel に登録します。
  2. 各カレンダーでサービスアカウントの `client_email` に「予定の変更」権限を付けます。
  3. Solo Booth は 1〜5 の 5 カレンダーを用意し、それぞれの ID を環境変数へ登録します。
  4. 予約済み枠は Google Calendar の `freeBusy` 結果に基づいて UI に反映されます。

4. メール送信

予約確定時は Resend から確認メールを送信します。

メール本文には施設名、日時、人数、利用目的、必要に応じて部屋番号、キャンセルリンクが含まれます。

キャンセルリンクは `cancel.html#...` 形式で送信されます。

件名:
【CROSSPORT MSZ】予約確認 - {施設名} / {日付} {開始}-{終了}

5. 障害切り分け

予約が失敗する

  • Vercel Logs で `[book] request failed` を検索する
  • `stage` と `errorMessage` を確認する
  • `SITE_URL is not configured` なら env を追加する
  • `Calendar API 404` ならカレンダー ID を見直す

メールが届かない

  • Resend Dashboard の送信ログを確認する
  • `RESEND_FROM_EMAIL` が検証済みか確認する
  • Vercel の `RESEND_API_KEY` が正しいか確認する

Solo Booth が満室にならない

  • `GCAL_ID_SOLO_BOOTH_1`〜`5` が全部設定されているか確認する
  • 5 カレンダーすべてにサービスアカウント共有があるか確認する

空き表示がおかしい

  • `/api/availability?facility=...&date=YYYY-MM-DD` の JSON を直接確認する
  • Google Calendar 側のイベントが「予定あり」か確認する

6. 運用メモ

  • 当日分の TIME SLOT は現在時刻を過ぎると予約不可になります。
  • カード上の `次の空き` は現在時刻 + 30 分基準で表示されます。
  • 16:30 以降は翌日の空き状況をカードに表示します。
  • 予約確認メールのキャンセルリンクから予約削除が可能です。
