# Phase 5: テスト・デプロイ

## 概要

全施設の予約フローを E2E で検証し、本番環境にデプロイする。

## タスク

### E2E テスト

- [ ] Event Space: 日付選択 → スロット選択 → 予約送信 → メール受信 → キャンセル
- [ ] Meeting Room: 同上
- [ ] Solo Booth: 同上
- [ ] 競合テスト: 同一スロットへの同時予約で片方が 409
- [ ] キャンセル済みスロットの再予約が可能であること
- [ ] 不正トークンでのキャンセル試行が 403 で拒否されること
- [ ] レート制限の動作確認

### レスポンシブ

- [ ] デスクトップ (1280px): 3カラムカード + フォーム
- [ ] タブレット (768px): 2カラムカード
- [ ] モバイル (375px): 1カラムカード、フォームのタッチ操作

### ブラウザ

- [ ] Chrome / Safari / Firefox で動作確認
- [ ] iOS Safari でのモバイル動作確認

### 本番デプロイ

- [ ] Vercel Preview デプロイで最終確認
- [ ] DNS / カスタムドメイン設定（必要に応じて）
- [ ] 本番デプロイ
- [ ] GitHub Pages ワークフロー無効化確認
